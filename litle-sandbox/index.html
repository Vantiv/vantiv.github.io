Accessing the Litle Sandbox

The Litle Sandbox functions as an emulator of our production environment.  You send your transactions to our sandbox environment the exact same way you’d send them to our certification or  production systems.  The sandbox validates the transaction format and approves it if it matches our requirements.  The Sandbox server makes it easy to get started using our SDK’s.  There are no forms to fill out, no credentials required, and no special access needed!

The Litle Sandbox environment is easily accessible to all users, simply configure your SDK and choose the “sandbox” option.  The sandbox is the default option, so it will be chosen automatically if you haven't chosen a destination.

URL: https://www.testlitle.com/sandbox/communicator/online

Note: Selecting the URL above will result in an error, the sandbox expects xml in the HTTP POST body.

The sandbox uses the request to calculate the response.  This is done so that any response that Litle's systems can generate can be retrieved through the sandbox as well.  The last 3 digits of a request field will be used to determine the response reason code.

Authorizations, Sales and all eCheck transactions use the amount.  To receive an approval, use an amount that ends with 000.

Captures, Voids, Credits and Reversals use the litleTxnId.  To receive an approval, use a litleTxnId that ends with 000.
