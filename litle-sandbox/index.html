Accessing the Litle Sandbox

The Litle Sandbox functions as an emulator of our production environment.  You send your transactions to our sandbox environment the exact same way you’d send them to our certification or  production systems.  The sandbox validates the transaction format and approves it if it matches our requirements.  The Sandbox server makes it easy to get started using our SDK’s.  There are no forms to fill out, no credentials required, and no special access needed!

The Litle Sandbox environment is easily accessible to all users, simply configure your SDK and choose the “sandbox” option.  The sandbox is the default option, so it will be chosen automatically if you haven't chosen a destination.

URL: https://www.testlitle.com/sandbox/communicator/online

Note: Selecting the URL above will result in an error, the sandbox expects xml in the HTTP POST body.

This environment is a production simulator. It can only handle online XML and needs at least version 8. It has no state, and doesn't support tied transactions by actually tying the transaction.

The sandbox uses the request to calculate the response.  This is done so that any response that Litle's systems can generate can be retrieved through the sandbox as well.

For sale and auth, the last 3 digits of the credit card will be used for the response reason code. Some sample numbers/responses include:

Desired Response Credit Card Number
000: Approved 4470330769941000
010: Partially Approved 4658512425423010
100: Processing Network Unavailable 4886883711815100
101: Issuer Unavailable 4215176886320101
110: Insufficient Funds 4488282659650110

Other numbers in the credit card can be used to emulate a variety of features, including avs, cvv, auto au, etc

Feature Credit Card Number Varient Details
Account Updater 4100117890123000
Vault/Tokenization 4100280190123000 The green characters specify the feature (vault). The red characters specify the token response code. The blue digits specify the response reason code.
Enhanced Authorizations 4100322111199000
Address Verification 4215176886320101
Card Security Code Validation 4100500000000000

On tied transactions, the last 3 digits of the litleTxnId are used as the response reason code.

This environment is on the public internet and requires no username or password.

It is typically used by merchants as their first test environment, while their merchant profile is being created. Some of our merchants continue to use it during development and maintenance as it provides consistent response codes.
